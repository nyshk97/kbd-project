# 意思決定の記録

## プロジェクト目的：自分用の最高のキーボード

- このプロジェクトは自分用の最高のキーボードを作ることを目的にする
- 量産・販売・第三者向けキット化は目的にしない
- 一般的な使いやすさより、自分の手・作業環境・好みに合うことを優先する
- 部品単価、入手性、製造容易性は考慮するが、量産時の原価最適化やサプライチェーン安定性は優先しない
- 販売向けの認証、パッケージング、ユーザー向け組み立て性、販売後サポートのための設計は対象外にする
- BLE無線モジュールの使用地ごとの法令適合は、販売有無と別に部品選定時に確認する

## ファームウェア：ZMK

- 完全無線・省電力・Auto Mouseレイヤー・エンコーダー対応が全部揃う
- QMKは有線前提のため非対応

## 接続方式：BLEドングル

- BLE直結はMacとの相性問題・スリープ復帰（約5秒）の懸念あり
- BLEドングルはUSB常時接続でほぼ即時復帰
- ドングル挿しっぱなしへの抵抗感は小さいと判断

## スイッチ：Choc v1 Red

- ロープロファイルで最薄、Macの打鍵感に最も近い
- カスタムビルド実績が豊富でErgogenフットプリントも充実
- キーキャップ自作前提のためステム規格の制約を受けない

## トラックボールセンサー：PMW3610

- PMW3360はZMK非対応
- PMW3610はZMKドライバ公開済み・超低消費電力・バッテリー持ち数週間単位
- ブレイクアウトボードが市販されており回路設計不要
- センサーボードはSideraKB PMW3610 PCB Rev 2.x系互換を前提にする
- トラックボール径は34mmで固定する
- Ergogen初期段階では34mm円、46mm keepout円、23.5mm×31.5mm基板矩形を配置して、支持機構やケース詳細は後段で詰める

## エンコーダー：EVQWGD001

- 縦回転（スクロールホイール型）が要件
- EC11は水平回転のため非対応
- Chocキーボードとの相性良好、ZMK分割キーボードでの実績あり
- エンコーダーは左手（central）側のみのためsplit encoder問題を回避

## PCB設計方針：Ergogen + FreeRouting + KiCad

- 要件の組み合わせ（ロースタッガード×分割×トラックボール×エンコーダー×無線）を満たすOSSのPCBが存在しない
- フルスクラッチ必須だがGUI作業を最小化する方針
- Ergogenでフットプリント自動配置、FreeRoutingで自動配線、KiCadは最終調整のみ
- 最初のErgogen成果物はキー配置・外形・1:1印刷用アウトラインまでに限定する
- KLEはキー数・キー順・ラベルの基準として固定し、Ergogen上の物理座標は紙プロトタイプ後に調整可能とする

## ケース（フェーズ1）：3Dプリント

- 動作確認フェーズは速度・コスト優先
- DMM.make or JLCPCBに発注
- フェーズ2の素材（アルミ削り出し等）はフェーズ1完了後に検討

## キーキャップ

- Choc v1対応の印字ありキャップは市販品の選択肢が少ない
- フェーズ1は印字なしMBK（1U）+ Enterのみ3Dプリント自作で割り切り
- フェーズ2で印字方法を本格検討
